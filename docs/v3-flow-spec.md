## 1. Reading (리딩)

### 1-1. 모듈 구성

리딩 모듈 1개 = **총 35문제**, 아래 순서대로 진행:

| 순서 | 유형 | 세트 수 | 세트당 문제 수 | 소계 |
|---|---|---|---|---|
| 1 | Fill in the Blank (빈칸채우기) | 2세트 | 10문제 | 20문제 |
| 2 | Daily 1 | 2세트 | 2문제 | 4문제 |
| 3 | Daily 2 | 2세트 | 3문제 | 6문제 |
| 4 | Academic Reading | 1세트 | 5문제 | 5문제 |
| | | | **합계** | **35문제** |

### 1-2. 실전풀이 진행 흐름 ✅ 확정

#### 시작
- 과제 대시보드(taskDashboardScreen)에서 **실전풀이 버튼** 클릭
- 안내 팝업 없이 **바로 빈칸채우기 세트1** 시작
- 동시에 **20분 타이머 카운트다운** 시작

#### 진행 순서
```
FB세트1(Q1-10) → FB세트2(Q11-20) → Daily1세트1(Q21-22) → Daily1세트2(Q23-24)
→ Daily2세트1(Q25-27) → Daily2세트2(Q28-30) → Academic(Q31-35)
```

#### 문제 번호 표시 ✅ 확정
- **빈칸채우기**: 한 페이지에 10문제가 모두 표시 → `Question 1-10 of 35`, `Question 11-20 of 35`
- **Daily1 / Daily2 / Academic**: 1문제씩 표시 → `Question 21 of 35`, `Question 22 of 35` …

#### 이동 방식 ✅ 확정
- **Next / Back 버튼**으로 자유 이동
- 1번 문제(FB세트1)에서는 **Back 버튼 없음**
- 35번 문제(Academic 마지막)에서는 **Next 대신 Submit 버튼** 표시
- 2번~34번 문제에서는 **Next + Back 모두 표시**

#### 세트 전환 ✅ 확정
- 세트 간 전환 시 **알림 없음**
- 학생이 **Next 버튼을 눌러야** 다음 세트로 이동
- 빈칸채우기는 한 페이지에 10문제이므로 Next 1회 = 세트 전체 이동

#### 역방향 이동 ✅ 확정
- FB세트2(Q11-20)에서 Back → FB세트1(Q1-10) 화면 전체 표시
- 이미 지나간 세트로 자유롭게 돌아갈 수 있음

#### 답안 기억 ✅ 확정
- 이전 문제로 돌아가면 **선택/입력한 답이 그대로 남아있음**
- 한번 선택한 보기는 **절대 자동 해제되지 않음** (학생이 직접 다른 보기를 선택해야 변경)

#### Review Panel ✅ 확정
- 상단 헤더에 **Review 버튼**
- 누르면 **1번~35번 전체 리스트** 표시
- 각 문제 옆에 **Answered / Not Answered** 상태 표시
- 문제를 클릭하면 **해당 문제로 바로 이동** (이미 지나간 세트 포함)
- Review Panel에서 이동 후 Next를 누르면 **해당 위치의 다음 문제로** 이동 (이전에 풀던 곳이 아님)
- **보조 기능** — 사용하지 않아도 풀이에 지장 없음 (실제 시험에 있는 기능이므로 구현)

#### 시간 초과 ✅ 확정
- 20분 타이머가 0이 되면 **즉시 자동 제출** (대기/경고 없음)
- 그때까지 입력한 답만 채점

#### 완료 후 화면 ✅ 확정
- Submit 버튼 클릭 또는 자동 제출 → **별도 결과 화면 없음**
- 바로 **taskDashboardScreen으로 복귀**
- 채점 결과는 taskDashboardScreen의 **우측 채점 대시보드**에서 확인
- 팝업 시점(시작 전/종료 시)은 핵심 플로우 확정 후 나중에 추가 가능

#### 저장 ✅ 확정
- 실전풀이 완료 시 `initial_record`에 저장 (최초 1회, 이후 변경 불가)
- 저장 내용: week/day, 과제명, **각 문제별 선택 답안 + 채점 결과**, 총 정답 수, 레벨

#### 레벨 변환표 (Reading, 총 35문제) ✅ 확정

| 정답 수 | 레벨 |
|---------|------|
| 0~3 | 1.0 |
| 4~6 | 1.5 |
| 7~10 | 2.0 |
| 11~13 | 2.5 |
| 14~17 | 3.0 |
| 18~20 | 3.5 |
| 21~24 | 4.0 |
| 25~27 | 4.5 |
| 28~30 | 5.0 |
| 31~32 | 5.5 |
| 33~35 | 6.0 |

### 1-3. 다시풀기 ✅ 확정

- 실전풀이와 **완전히 동일한 흐름** (같은 35문제, 같은 순서, 같은 20분 타이머, 같은 이동/Review 방식)
- 유일한 차이: 저장 위치가 `initial_record` → `current_record`
- 상세 규칙은 v3-design-spec.md 2-3 참조

### 1-4. 브라우저 새로고침/닫기 시 동작 ✅ 확정

- 현재 확정: 풀이 중 브라우저를 닫거나 새로고침하면 **풀이 내용이 사라짐** (처음부터 다시 풀어야 함)
- **이어풀기 기능(중간 저장 후 재접속 시 복원)은 핵심 플로우가 전부 완성된 뒤에 재검토 예정**
  - 구현 시 localStorage 활용, 기존 플로우를 건드리지 않고 추가 가능
  - 단, 다른 기기/브라우저에서는 복원 불가 (localStorage 특성)

## 2. Listening (리스닝)

### 2-1. 모듈 구성 ✅ 확정

리스닝 모듈 1개 = **총 32문제**, 아래 순서대로 진행:

| 순서 | 유형 | 세트 수 | 세트당 문제 수 | 소계 |
|---|---|---|---|---|
| 1 | Response (응답고르기) | 1세트 | 12문제 | 12문제 |
| 2 | Conversation (대화) | 3세트 | 2문제 | 6문제 |
| 3 | Announcement (공지사항) | 3세트 | 2문제 | 6문제 |
| 4 | Lecture (렉쳐) | 2세트 | 4문제 | 8문제 |
| | | | **합계** | **32문제** |

### 2-2. 리딩과의 핵심 차이점 ✅ 확정

| 항목 | 리딩 | 리스닝 |
|---|---|---|
| 타이머 | 모듈 전체 20분 | 문제별 개별 타이머 (20초/30초) |
| 이동 방식 | Next/Back 자유 이동 | **순방향만** (Back 없음, 되돌아가기 불가) |
| Review Panel | 문제 클릭 시 해당 문제로 이동 | **열람용만** (클릭해도 이동 안 함) |
| 오디오 | 없음 | 있음 (수동 재생, 1회만) |

### 2-3. 실전풀이 진행 흐름 ✅ 확정

#### 공통 규칙

- **자동 규칙 최소화**: 자동 재생 없음, 자동 넘김 없음
- **모든 이동은 학생이 Next 클릭**
- **오디오는 한 번만 재생 가능** (재재생 불가)
- 오디오 재생은 **학생이 재생 버튼을 눌러야** 시작 (자동 재생 절대 안 함)
- **Back 버튼 없음** — 한 번 넘어가면 이전 문제로 되돌아갈 수 없음
- 문제 번호: **Question 1 of 32** ~ **Question 32 of 32** (전부 1문제씩 표시)
- 마지막 문제(32번)에서 **Submit 버튼** 표시
- 완료 후 별도 결과 화면 없이 **과제 화면(taskDashboardScreen)으로 복귀**

#### Next 버튼과 오디오의 관계 ✅ 확정

- 오디오 **재생 전**: Next 버튼 보임 (오디오를 듣지 않고 넘기기 가능)
- 오디오 **재생 중**: Next 버튼 숨김 (중간에 끊고 넘기기 방지)
- 오디오 **재생 완료**: Next 버튼 다시 보임

#### Response (응답고르기) 흐름 ✅ 확정

```
문제 화면 표시 (재생 버튼 + 보기 4개 함께)
  → 보기는 비활성화 상태 (흐리게)
  → 학생이 재생 버튼 클릭 → 오디오 재생 (재생 중 Next 숨김)
  → 오디오 끝남 → 보기 활성화 + 20초 카운트다운 시작 + Next 표시
  → 20초 경과 → 보기 다시 비활성화 (선택 불가)
  → 학생이 Next 클릭 → 다음 문제
```
- 이 흐름을 12문제 반복
- 12문제 완료 후 → Conversation 유형으로 이동

#### Conversation / Announcement / Lecture 공통 흐름 ✅ 확정

```
[오디오 재생 화면] (제목 + 사진 + 재생 버튼)
  - 헤더에 타이머, 문제 카운트(Question X of 32) 표시 안 함
  - 학생이 재생 버튼 클릭 → 나레이션 재생 → 메인 오디오 재생 (재생 중 Next 숨김)
  - 오디오 끝남 → Next 표시 → 학생이 Next 클릭

[문제 화면] (왼쪽: 사진 / 오른쪽: 문제 + 보기 4개)
  - 문제 화면 표시 순간 카운트다운 시작 (Conver/Announce: 20초, Lecture: 30초)
  - 시간 경과 → 보기 비활성화
  - 학생이 Next 클릭 → 다음 문제 (같은 세트 내)
  - 세트의 마지막 문제 → Next 클릭 → 다음 세트의 오디오 재생 화면으로 이동
```

#### 나레이션 규칙 ✅ 확정

| 유형 | 나레이션 |
|---|---|
| Conversation | 하드코딩 — "Listen to a conversation" (매번 동일, GitHub 호스팅) |
| Announcement | 세트별 개별 등록 (매번 다름) |
| Lecture | 세트별 개별 등록 (매번 다름) |

#### Review Panel (리스닝) ✅ 확정

- 상단 헤더에 **Review 버튼**
- 누르면 **1번~32번 전체 리스트** 표시
- 각 문제 옆에 **Answered / Not Answered** 표시
- **열람 전용** — 문제를 클릭해도 해당 문제로 이동하지 않음

### 2-4. 다시풀기 ✅ 확정

- 실전풀이와 **완전히 동일한 흐름** (같은 32문제, 같은 순서, 같은 타이머, 같은 이동 방식)
- 유일한 차이: 저장 위치가 `initial_record` → `current_record`

### 2-5. 브라우저 새로고침/닫기 시 동작 ✅ 확정

- 리딩과 동일 — 풀이 내용 사라짐, 이어풀기는 핵심 플로우 완성 후 재검토 예정

### 2-6. 레벨 변환표 (Listening, 총 32문제) ✅ 확정

| 정답 수 | 레벨 |
|---------|------|
| 0~2 | 1.0 |
| 3~5 | 1.5 |
| 6~8 | 2.0 |
| 9~11 | 2.5 |
| 12~15 | 3.0 |
| 16~18 | 3.5 |

## 3. Writing (라이팅)

### 3-1. 모듈 구성 ✅ 확정

라이팅 모듈 1개 = **총 12문제 카운트**, 아래 순서대로 진행:

| 순서 | 유형 | 세트 수 | 문제 수 | 제한 시간 |
|---|---|---|---|---|
| 1 | Arrange (단어배열) | 1세트 | 10문제 | 6분 50초 |
| 2 | Email (이메일 작성) | 1세트 | 작성 1건 | 7분 |
| 3 | Discussion (토론 작성) | 1세트 | 작성 1건 | 10분 |

- 문제 번호: Arrange Q1~Q10, Email Q11, Discussion Q12

### 3-2. 실전풀이 진행 흐름 ✅ 확정

#### 공통 규칙

- **유형별 개별 타이머** (모듈 전체 타이머 없음)
- 시간 초과 시 → **입력/선택 비활성화** → 학생이 **Next 버튼을 눌러야** 다음으로 이동 (자동 넘김 없음)
- **Review Panel 없음** (라이팅에는 리뷰 패널이 없음)
- 문제 번호: **Question 1 of 12** ~ **Question 12 of 12**
- 완료 후 별도 결과 화면 없이 **과제 화면(taskDashboardScreen)으로 복귀**

#### Arrange (단어배열) 흐름 ✅ 확정

```
문제 화면 표시 (드래그 & 드롭으로 단어 배열)
  → Next/Back 버튼으로 자유 이동 (리딩과 동일)
  → 1번 문제에 Back 없음, 10번 문제에 Next 대신 → Email로 이동하는 버튼
  → 유형 전체 제한 시간: 6분 50초
  → 시간 초과 → 입력 비활성화 → Next 눌러서 이동
```

#### Email (이메일 작성) 흐름 ✅ 확정

```
작성 화면 표시 (컴포넌트에 구현된 화면 그대로)
  → 텍스트 직접 입력, 편집 도구(Cut/Paste/Undo/Redo), 단어수 카운트
  → 유형 제한 시간: 7분
  → 시간 초과 → 입력 비활성화 → Next 눌러서 이동
  → 상단 버튼(Next) 클릭 → 자동으로 DB 저장 + Discussion 화면으로 이동
```

#### Discussion (토론 작성) 흐름 ✅ 확정

```
작성 화면 표시 (컴포넌트에 구현된 화면 그대로)
  → Email과 동일한 입력/편집 방식
  → 유형 제한 시간: 10분
  → 시간 초과 → 입력 비활성화 → Submit 눌러서 완료
  → 상단 버튼(Submit) 클릭 → 자동으로 DB 저장 + 과제 화면으로 복귀
```

### 3-3. 저장 규칙 ✅ 확정

**실전풀이 (initial_record):**
- Arrange: 채점 결과 저장
- Email: 작성 내용이 `writing_email_text` 별도 컬럼에 **최초 1회만 저장** + initial_record에 포함
- Discussion: 작성 내용이 `writing_discussion_text` 별도 컬럼에 **최초 1회만 저장** + initial_record에 포함

**다시풀기 (current_record):**
- Arrange: 채점 결과가 current_record에 저장
- Email / Discussion: **current_record에 작성 내용 포함 저장 + .txt 파일 자동 다운로드 동시 수행**
  - `writing_email_text`, `writing_discussion_text` 별도 컬럼은 변경하지 않음 (최초 1회 원칙 유지)
  - 해설보기에서 "내가 쓴 글" 표시를 위해 current_record에 포함 필요

### 3-4. 채점 대시보드 표시 ✅ 확정

```
[채점 결과 - 실전풀이]
├── Arrange: 8/10 정답
├── Email: 작성 완료 (108 words)
├── Discussion: 작성 완료 (95 words)
```

- **레벨 변환표 없음** — Arrange만 정답 수 표시, Email/Discussion은 "작성 완료" 표시

### 3-5. 다시풀기 ✅ 확정

- 화면은 실전풀이와 **완전히 동일**
- 저장 위치만 다름 (initial_record → current_record)
- Email/Discussion은 추가로 .txt 파일 다운로드

### 3-6. 브라우저 새로고침/닫기 시 동작 ✅ 확정

## 4. Speaking (스피킹)

### 4-1. 모듈 구성 ✅ 확정

스피킹 모듈 1개 = **총 11문제**, 아래 순서대로 진행:

| 순서 | 유형 | 세트 수 | 세트당 문제 수 | 소계 |
|---|---|---|---|---|
| 1 | Repeat (따라말하기) | 1세트 | 7문제 | 7문제 |
| 2 | Interview (인터뷰) | 1세트 | 4문제 | 4문제 |
| | | | **합계** | **11문제** |

### 4-2. 실전풀이 진행 흐름 ✅ 확정

#### 핵심 특성: 완전 자동 시퀀스

- 스피킹은 **학생이 누르는 버튼이 하나도 없음**
- 컴포넌트 내부 로직이 전체 흐름을 자동으로 진행
- Repeat: 오디오 재생 → 비프음 → 녹음 타이머 → 다음 문제 (7문제 반복)
- Interview: 비디오 재생 → 녹음 타이머 → 다음 문제 (4문제 반복)

#### 시작 ✅ 확정

- 과제 화면(taskDashboardScreen)에서 **실전풀이 버튼** 클릭
- 바로 **Repeat 인트로 화면** 시작 + 나레이션 자동 재생

#### 진행 순서 ✅ 확정

```
Repeat 인트로(자동) → Repeat Q1~Q7(자동) → Interview 인트로(자동) → Interview Q8~Q11(자동) → 완료
```

#### Repeat → Interview 전환 ✅ 확정

- Repeat 7번째 문제 녹음 끝나면 → **바로 Interview 인트로 화면 + 나레이션 자동 재생**
- 학생 입장에서는 자연스럽게 이어짐
- **내부적으로는 Repeat 컴포넌트 완전 정리(오디오 중지, 타이머 해제 등) 확인 후 Interview 시작** (순서 보장, 오디오 겹침 방지)

#### 이동 방식 ✅ 확정

- **Next 버튼 없음, Back 버튼 없음** — 전부 자동 진행
- Review Panel 없음

#### 문제 번호 표시 ✅ 확정

- **"Question X of 11"** 형식 (Q1~Q11)
- **미표시 화면**: Repeat 인트로, Interview 인트로
- **표시 화면**: 그 외 모든 화면 (상황 나레이션, 오디오 재생, 녹음 등)

#### 타이머 ✅ 확정

- **전체 모듈 타이머 없음**
- 각 문제별 내부 타이머만 존재 (컴포넌트 자체 관리)
- Repeat: 문제별 응답 시간 (데이터에서 개별 설정)
- Interview: 45초 고정

#### 완료 후 화면 ✅ 확정

- Interview 마지막 문제 녹음 끝나면 → **taskDashboardScreen으로 바로 복귀**
- 시작 전/종료 시 팝업은 후순위 (핵심 플로우 확정 후 추가 가능)

### 4-3. 저장 ✅ 확정

- initial_record에 **완료 여부만** 기록 (채점 결과 없음)
- 대시보드 표시: "따라말하기: 완료 ✅ / 인터뷰: 완료 ✅"
- **레벨 변환표 없음** (스피킹은 자동 채점 불가)

### 4-4. 다시풀기 ✅ 확정

- 실전풀이와 **완전히 동일한 흐름** (같은 11문제, 같은 자동 시퀀스)
- current_record에 완료 여부만 기록
- 녹음 파일 별도 제출 없음 (v3-design-spec.md 2-3-2 참조)

### 4-5. 브라우저 새로고침/닫기 시 동작 ✅ 확정

- 다른 섹션과 동일 — 풀이 내용 사라짐, 이어풀기는 핵심 플로우 완성 후 재검토 예정
